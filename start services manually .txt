1. To install on client

FIRST TIME (installs Node, npm, cloudflared, npm packages, build, task):
  Open PowerShell as Administrator
  cd E:\WINDAL\APPTEST
  powershell -ExecutionPolicy Bypass -File .\setup-client.ps1 -OracleConnectString "XE" -OraclePrimaryPassword "CLIENT_PASSWORD_HERE"

If you see "node is not recognized" or "npm is not recognized":
  - Node is not installed or not on PATH. Run setup-client.ps1 above (it installs Node LTS via winget), then CLOSE PowerShell and open a NEW Administrator window, then cd to APPTEST again.
  - Or install Node.js 18+ LTS from https://nodejs.org/ (64-bit), reboot or open new PowerShell, verify:  node --version

To use Node in THIS session only (typical install paths):
  $env:Path += ";C:\Program Files\nodejs;${env:LOCALAPPDATA}\Programs\nodejs"
  node --version

If PowerShell says npm.ps1 "running scripts is disabled":
  Use the CMD shim (avoids ExecutionPolicy on npm.ps1):
    npm.cmd run server
  Or relax policy for your user (one-time):
    Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

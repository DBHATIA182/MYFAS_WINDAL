Windal — Client installation (D:\WINDAL)

This guide is for installing Windal Accounting (APPTEST web app + API) on a client PC. Example paths use D:\WINDAL; you may use another drive if needed.

1. Overview and folder layout

D:\WINDAL\
  apptest\                         ← Application (Git clone target)
    server.cjs, package.json, SRC, scripts (.ps1, .cmd)
    connection.config.json         ← Per-client Oracle + client key (secrets)
    config.yml                     ← Cloudflare tunnel ingress
    xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx.json   ← Tunnel credentials
  oracle_bridge\
    instantclient_23_0\            ← Oracle Instant Client (required)
  tnsnames.ora, sqlnet.ora           ← Optional TNS (if used)

The API reads Oracle from ..\oracle_bridge\instantclient_23_0 relative to apptest.

2. Prerequisites checklist

  • Windows 10 or 11 (64-bit).
  • PowerShell as Administrator for setup and scheduled tasks.
  • Internet for first install (Git clone, npm, winget).
  • Oracle database reachable from this PC; Instant Client at D:\WINDAL\oracle_bridge\instantclient_23_0.
  • Cloudflare account access for zone fasaccountingsoftware.in (tunnel).

3. Install Git for Windows (one time per PC)

Git is required to clone the app the first time and to pull updates later.

3.1 Download and install

  1. Open https://git-scm.com/download/win
  2. Download 64-bit Git for Windows Setup.
  3. Run the installer → accept defaults (ensure Git from the command line and also from 3rd-party software is selected).
  4. Finish and open a new PowerShell window.

3.2 Verify Git

git --version

Expected: git version 2.x.x

Optional — winget:
winget install --id Git.Git -e --accept-source-agreements --accept-package-agreements
Close PowerShell and open a new window, then run git --version again.

3.3 Configure Git (recommended once per PC)

git config --global user.name "Your Name"
git config --global user.email "you@company.com"

For private GitHub repos, use a Personal Access Token as password when Git prompts during clone/pull, or set up SSH keys.

4. First-time install — get the app from GitHub

Repository: https://github.com/DBHATIA182/MYFAS_WINDAL.git
Branch: main

4.1 Option A — Fresh clone (recommended for new client PC)

  1. Create parent folder if needed: 
    mkdir D:\WINDAL -Force
  2. Clone the repository: 
    cd D:\WINDAL
git clone https://github.com/DBHATIA182/MYFAS_WINDAL.git apptest
cd D:\WINDAL\apptest
  3. If you have an old copy of the app (not from Git), copy these files from the old folder into D:\WINDAL\apptest after clone:
    • connection.config.json
    • config.yml
    • Tunnel credentials file: <tunnel-uuid>.json (same name as in config.yml)

4.2 Option B — Automated first-time Git install script

Copy git_client_installation.ps1 to D:\WINDAL (parent folder, not inside apptest). Edit paths inside the script if your drive differs.

cd D:\WINDAL
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass -Force
.\git_client_installation.ps1

This script: renames existing APPTEST to APPTEST_OLD if needed, clones the repo, copies connection.config.json / tunnel files from backup, runs npm ci and npm run build.

4.3 What NOT to copy from dev PC

  • Do not copy node_modules — install on client with npm ci.
  • dist is optional — setup rebuilds the UI.

5. Install Node.js and Cloudflare (cloudflared)

5.1 Check if already installed

node --version
npm --version
cloudflared --version

Need Node.js 18 or newer (LTS recommended).

5.2 Automatic (recommended) — via setup-client.ps1

setup-client.ps1 installs Node LTS and cloudflared using winget when missing (Step 7).

5.3 Manual install with winget (Administrator)

winget install --id OpenJS.NodeJS.LTS -e --accept-source-agreements --accept-package-agreements
winget install --id Cloudflare.cloudflared -e --accept-source-agreements --accept-package-agreements

Close PowerShell, open a new window, verify commands again.

5.4 Manual install from websites

  • Node.js: https://nodejs.org/ — download LTS, install with defaults.
  • cloudflared: https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/downloads/ — Windows 64-bit.

5.5 Cloudflare login (one time per PC)

cloudflared tunnel login

Browser opens → log in → select zone fasaccountingsoftware.in.

6. Configure connection.config.json

Edit D:\WINDAL\apptest\connection.config.json for this client:

  • clientName / defaultClientKey — e.g. dal-pushkar (use hyphens in DNS, not underscores)
  • oracle.primary — user, password, connect string (e.g. XE or HOST:1521/SERVICE)

Setup script can write this file for you (next step).

7. Run setup-client.ps1 (one-time setup)

Open PowerShell as Administrator:

cd D:\WINDAL\apptest

powershell -ExecutionPolicy Bypass -File .\setup-client.ps1 `
  -ClientKey "dal-pushkar" `
  -OraclePrimaryUser "DAL" `
  -OraclePrimaryPassword "YOUR_PASSWORD" `
  -OracleConnectString "XE" `
  -OracleSecondaryUser "DAL" `
  -OracleSecondaryPassword "YOUR_PASSWORD"

This typically:

  • Updates connection.config.json
  • Installs Node / cloudflared if missing
  • Runs npm install and npm run build
  • Creates scheduled task FAS-dal-pushkar-API (API auto-start on reboot)
After Node install, if node is not found, close PowerShell and open a new window.

8. Cloudflare tunnel (config.yml)

If tunnel dal-pushkar already exists in Cloudflare, you do not create it again. On this PC you need:

  • D:\WINDAL\apptest\config.yml
  • D:\WINDAL\apptest\<tunnel-uuid>.json (credentials)

8.1 Example config.yml

tunnel: YOUR-TUNNEL-UUID-HERE
credentials-file: ./YOUR-TUNNEL-UUID-HERE.json

ingress:
  - hostname: dal-pushkar.fasaccountingsoftware.in
    service: http://localhost:5174
  - hostname: dal-pushkar-api.fasaccountingsoftware.in
    service: http://localhost:5001
  - service: http_status:404

8.2 Regenerate config on client (same tunnel name)

cloudflared tunnel login
cd D:\WINDAL\apptest
powershell -ExecutionPolicy Bypass -File .\setup-new-client-tunnel.ps1 -ClientKey "dal-pushkar" -SkipClientSetup

8.3 List tunnels

cloudflared tunnel list

11. Updating the app from Git (routine)

When developers publish changes to GitHub, update the client PC:

11.1 Stop services first

cd D:\WINDAL\apptest
powershell -ExecutionPolicy Bypass -File .\stop-apptest-services.ps1 `
  -AppRoot "D:\WINDAL\apptest" -StopScheduledTasks -ReleaseApiPort5001 -ReleasePorts5174

11.2 Pull and rebuild

cd D:\WINDAL\apptest
powershell -ExecutionPolicy Bypass -File .\update-from-git.ps1 -Branch main

This runs: git fetch, git pull origin main, npm ci, npm run build.

11.3 Start services again

powershell -ExecutionPolicy Bypass -File .\start-apptest-services.ps1 -AppRoot "D:\WINDAL\apptest"

11.4 Manual Git commands (if scripts fail)

cd D:\WINDAL\apptest
git fetch origin
git checkout main
git pull origin main --autostash
npm.cmd ci
npm.cmd run build
Files Git must NOT overwrite: Keep local copies of connection.config.json, config.yml, and tunnel *.json. If pull changes them, restore from backup.

12. One-click desktop update

Use the desktop shortcut Update APPTEST (or run):

cd D:\WINDAL\apptest
powershell -ExecutionPolicy Bypass -File .\run-desktop-update.ps1 -AppRoot "D:\WINDAL\apptest" -Branch main

Log file: D:\WINDAL\apptest\logs\desktop-update.log

Steps: stop services → git pull + npm ci + build → start services.

13. Troubleshooting

ProblemAction
node / npm not foundReopen PowerShell after install; or add C:\Program Files\nodejs to PATH.
git not foundInstall Git for Windows; reopen PowerShell.
Not a Git repositoryRun first-time clone (Section 4) or git_client_installation.ps1.
git pull failedRun git status; stash local changes: git stash -u, pull, then git stash pop.
File locked during updateClose API/Vite/tunnel windows; run stop script with -StopScheduledTasks; do not open log files in Notepad while updating.
Oracle init errorConfirm D:\WINDAL\oracle_bridge\instantclient_23_0 exists; check TNS files.
Tunnel credentials missingPlace config.yml + <uuid>.json in apptest folder.
Wrong site on public URLVite must use port 5174; match config.yml ingress.

APPTEST — Client installation guide

This document covers copying the app to a client PC, Cloudflare tunnel configuration, running setup-client.ps1, optional scheduled tasks for auto-start, and how to start all services manually the first time.

1. Prerequisites

  • Windows 10 or 11 (64-bit).
  • Administrator PowerShell for setup and scheduled tasks.
  • Internet access for first-time npm install and optional winget installs (Node.js LTS, Cloudflare cloudflared).
  • winget available (Microsoft App Installer), unless you install Node and cloudflared yourself.

2. What to copy to the client machine

Copy the entire APPTEST application folder to any drive/path on the client (this folder is called <APP_ROOT> below). Example: C:\Apps\APPTEST.

2.1 Include these items

ItemPurpose
server.cjs, package.json, package-lock.jsonBackend and dependencies
connection.config.jsonClient name and Oracle settings (updated by setup)
setup-client.ps1One-time client install
config.ymlCloudflare tunnel ingress (edit per client)
run-backend.cmd, run-all-services.cmdLaunchers for Task Scheduler / manual use
setup-scheduled-task-all-services.ps1Optional: register startup task for API + frontend + tunnel
start-apptest-services.ps1Optional: open three PowerShell windows for manual dev-style start
SRC, public (if present)Frontend source and static assets

2.2 Do not copy

  • node_modules — the setup script runs npm install on the client.

2.3 Oracle Instant Client layout

The server expects Oracle Instant Client in a sibling folder next to the app parent, for example:

  • <APP_ROOT> = C:\Apps\APPTEST
  • C:\Apps\oracle_bridge\instantclient_23_0

Adjust paths to match server.cjs expectations on your deployment.

3. Cloudflare tunnel (config.yml)

Before relying on the tunnel, on the client PC:

  1. Create or use an existing tunnel in Cloudflare Zero Trust and note the tunnel UUID.
  2. Place the tunnel credentials JSON on the machine (commonly under C:\Users\<User>\.cloudflared\<tunnel-uuid>.json).
  3. Edit <APP_ROOT>\config.yml:
    • tunnel: — your tunnel UUID.
    • credentials-file: — full path to that JSON file.
    • ingress — hostnames for this client. Example for client key dal-rgind:
      • Web: dal-rgind.fasaccountingsoftware.in → http://localhost:5173
      • API: dal-rgind-api.fasaccountingsoftware.in → http://localhost:5001
  4. In Cloudflare DNS, route those hostnames to the same tunnel.
Scheduled task as SYSTEM: If the startup task runs under SYSTEM, ensure credentials-file points to a path SYSTEM can read (for example a folder under the app root with restricted permissions), or run the tunnel under a user account that owns the credentials file.

4. First-time install (setup-client.ps1)

  1. Open PowerShell as Administrator.
  2. Change to the app folder: 
    cd <APP_ROOT>
  3. Run setup (replace Oracle values with the client’s real connect string and passwords): 
    powershell -ExecutionPolicy Bypass -File .\setup-client.ps1 `
  -ClientKey YOUR_CLIENT_KEY `
  -OracleConnectString "HOST:1521/SERVICE_NAME" `
  -OraclePrimaryUser DAL `
  -OraclePrimaryPassword "YOUR_PASSWORD" `
  -OracleSecondaryUser DAL `
  -OracleSecondaryPassword "YOUR_PASSWORD" `
  -AutoStartMode task

The script typically: updates connection.config.json, installs Node/npm and cloudflared if missing (via winget), runs npm install, runs npm run build, and registers a startup task for the API only.

4.1 -AutoStartMode options

ValueMeaning
taskTask Scheduler runs run-backend.cmd at startup (API only). Task name: FAS-<client>-API.
nssmWindows Service via NSSM (requires nssm.exe in the app).
noneNo auto-start task.

5. Scheduled tasks (two approaches)

5.1 API only (default from setup-client.ps1)

  • Task name: FAS-<client>-API
  • Runs: run-backend.cmd → npm run server → log logs\server.log
  • Trigger: At system startup, as SYSTEM, highest privileges.

Use Task Scheduler (taskschd.msc) to confirm status and last run result.

5.2 API + Vite dev + Cloudflare tunnel (all three)

Use this when the client machine should bring up the web dev server and tunnel automatically after reboot.

  1. Ensure config.yml and tunnel credentials are correct.
  2. Open PowerShell as Administrator, cd <APP_ROOT>.
  3. Run: 
    powershell -ExecutionPolicy Bypass -File .\setup-scheduled-task-all-services.ps1
  • Default task name: FAS-<clientName>-AllServices (from connection.config.json).
  • Runs: run-all-services.cmd — starts minimized CMD windows for API, frontend, and tunnel.
  • Logs: logs\server.log, logs\frontend.log, logs\tunnel.log, plus logs\all-services.log for the launcher.
Important: If both FAS-<client>-API and FAS-<client>-AllServices are enabled, the API may start twice. Disable or delete the API-only task when using the “all services” task.

5.3 Optional: custom task name

.\setup-scheduled-task-all-services.ps1 -TaskName "FAS-dal-rgind-AllServices"

6. First-time manual run (all three services)

After npm install (and ideally npm run build from setup), use one of the following before relying on scheduled tasks.

6.1 Option A — Batch launcher (minimized windows)

From PowerShell or CMD, in <APP_ROOT>:

.\run-all-services.cmd

Starts API (port 5001 per app defaults), Vite on 0.0.0.0:5173, and cloudflared tunnel --config config.yml run. Check logs\ for output.

6.2 Option B — Interactive PowerShell windows

powershell -ExecutionPolicy Bypass -File .\start-apptest-services.ps1

Opens three separate PowerShell windows (good for watching live logs during first-time testing).

6.3 Option C — Three separate manual commands

Open three terminals, each:

cd <APP_ROOT>
npm run server
cd <APP_ROOT>
npm run dev -- --host 0.0.0.0 --port 5173
cd <APP_ROOT>
cloudflared tunnel --config config.yml run

7. Verification checklist

  • Task Scheduler: relevant task(s) show Ready and a successful last run (where applicable).
  • logs\server.log — API listening / no Oracle path errors.
  • Browser: client web hostname loads; API hostname responds as expected.
  • Tunnel: Cloudflare dashboard shows the connector healthy.

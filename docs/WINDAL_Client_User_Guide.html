Windal Accounting System

Client User Guide — Features & How to Use

For office staff · Reports, entries, drill-down navigation

1. Overview

Windal Accounting is a web application for trading and pulses businesses. It covers:

  • Final accounts — Trial Balance, Trading A/C, Profit & Loss, Balance Sheet
  • Ledgers — Account ledger, customer/supplier bills, broker outstanding, ageing
  • Stock — Stock summary, HSN sales/purchase reports
  • Sales — Order, dispatch challan, sale bill entry and printing, GSTR-1
  • Purchase — Purchase order, GRN, purchase bill, purchase list
  • Vouchers — Cash, bank, and journal entry (CV / BV / JV)
  • Production — Production entry and stock posting (TYPE PR)
  • Masters — Account master and item master

Open the system in Chrome or Edge on your PC or mobile using the URL provided by your administrator.

2. Log in and open a report

  1. Enter User name and Password (from your company user list).
  2. Select Company and Financial Year, then continue.
  3. On the main menu, click a module header (1–8) to expand it.
  4. Click the report or entry you need — it opens immediately.
Navigation: Use ← Back to return one screen. Use 🏠 Home to return to the menu. On any screen, tap ? (Help) for a short guide.

3. Trial Balance → Ledger → Voucher (step by step)

This is the most common way to check an account and see the original voucher behind a posting.

Example: check an account from Trial Balance

  1. Open Final Accounts → Trial Balance Report.
  2. Set As-of (ending) date and click Run Report.
  3. The list shows every account with opening, debit, credit, and closing balances.
  4. Click any account row (e.g. a party or expense account).
  5. The Ledger Account screen opens — all transactions for that account in the financial year.
  6. Click any transaction line on the ledger.
  7. The Voucher screen opens — all ledger lines posted on that voucher (type, date, number).
  8. Press ← Back to return to the ledger, then again to return to trial balance.

On some ledger lines (sale-related), you may also open Sale Bill print when the voucher is linked to a sale bill.

Trial Balance Summary (by annexure / schedule)

  1. Open Trial Balance Summary → set as-of date → Run Report.
  2. Summary shows one row per annexure (schedule).
  3. Click a summary row → account detail for that schedule.
  4. Click an account → Ledger → click a line → Voucher (same as above).

Trial Balance Date Wise

Set starting and ending dates. The report shows opening, period movement, and closing per account. Click an account row to open the ledger for that date range.

Same idea on other reports

  • Balance Sheet — click an account → ledger → voucher on line click.
  • Trading A/C / P&L — click highlighted amounts or accounts where enabled → ledger.
  • Voucher List — click a row → voucher detail lines.

5. Data entry screens (brief)

Sale Bill

Enter party, items, quantities, GST, etc. Save posts to SALE, LEDGER, STOCK, and BILLS. Use toolbar for Add, Edit, Delete, Print, Prev/Next. Rights from your user profile.

Purchase Bill

Enter supplier bill, lines with weights and GST. Save posts PURCHASE, STOCK, LEDGER, BILLS. Print from toolbar after save.

Dispatch Challan / Sales Order / Purchase Order / GRN

Standard document entry with party search, line items, Prev/Next between vouchers, List screen, and Print. Pending SO/PO can be picked on lines where shown.

Production Entry

Date, Sr.No., milling item, weight, qty, B/K/H, plant (required). Add production lines with item, prod %, qty, weight, shortage. Save posts PROD and STOCK (TYPE PR). List, Print, Prev/Next on toolbar.

Cash / Bank / Journal Voucher

CV, BV, JV entry with debit/credit lines. Save posts VOUCHER, LEDGER, BILLS. Use Voucher List to find and open existing vouchers.

6. Export and share reports

On most report screens the toolbar offers:

  • Pdf — save report as PDF file
  • Excel — download spreadsheet
  • Print — send to printer
  • WhatsApp — share PDF (useful on mobile)

7. Mobile use

  • Same URL as desktop; layout adapts to phone screen.
  • Swipe sideways on wide reports if needed.
  • Entry screens use search sheets for party and item selection.
  • Print opens the system print dialog; PDF saves to device or Downloads.

8. User rights

Your administrator controls which modules you can open, add, edit, or delete (stored in DAL.USERS — F2 sale bill, F5 production, F12 sales order, etc.). If you see Access Denied, ask your admin to update your rights.

9. In-app full PDF guide

After login, on the reports menu click ? (Help) and choose Download full user guide (PDF) for an up-to-date copy including every module.

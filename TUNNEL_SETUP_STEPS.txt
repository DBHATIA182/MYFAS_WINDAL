WINDAL — Cloudflare tunnel (client key: dal-pushkar)
=================================================

App folder: use the real path on that PC (e.g. D:\windal\apptest or E:\WINDAL\APPTEST).
Scripts do NOT require E: — only cd to your APPTEST folder first.

Public URLs (after DNS routes):
  https://dal-demo.fasaccountingsoftware.in          (Vite / web UI)
  https://dal-demo-api.fasaccountingsoftware.in      (Node API)

ONE-COMMAND SETUP (PowerShell — from your APPTEST folder):
  cd D:\windal\apptest
  powershell -ExecutionPolicy Bypass -File .\setup-new-client-tunnel.ps1 `
    -ClientKey "dal-demo" `
    -SkipClientSetup `
    -OraclePrimaryUser "DAL" `
    -OraclePrimaryPassword "DAL" `
    -OracleSecondaryUser "DAL" `
    -OracleSecondaryPassword "DAL" `
    -OracleConnectString "XE"

That script:
  - Runs setup-client.ps1 (unless -SkipClientSetup)
  - Creates or reuses Cloudflare tunnel "dal-demo"
  - Creates DNS CNAMEs dal-demo.fasaccountingsoftware.in and dal-demo-api.fasaccountingsoftware.in
  - Writes config.yml (web UI -> localhost:5174) and copies <tunnel-uuid>.json next to config.yml

Then start (3 terminals or use run-all-services.cmd):
  1) npm.cmd run server
  2) npm.cmd run dev -- --host 0.0.0.0 --port 5174
  3) cloudflared.exe tunnel --config .\config.yml run

DAL / Windal uses port 5174 for Vite (not 5173). config.yml ingress must match.

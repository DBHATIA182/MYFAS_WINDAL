WINDAL — Cloudflare tunnel setup (correct steps)
==============================================

Use ONE command at a time. Do not paste multi-line blocks backwards.

APP folder examples:
  E:\WINDAL\APPTEST
  D:\windal\apptest
  (same folder — drive letter does not matter)


STEP 0 — Install cloudflared (once per PC)
------------------------------------------
  winget install Cloudflare.cloudflared

Real binary is usually:
  C:\Program Files (x86)\Cloudflared\cloudflared.exe

Ignore C:\Windows\System32\cloudflared.exe (0-byte stub — breaks commands).


STEP 1 — Login to Cloudflare (once per PC)
------------------------------------------
  cd E:\WINDAL\APPTEST

  In PowerShell you MUST prefix with .\ :
  .\setup-windal-tunnel.cmd login

Or:
  powershell -ExecutionPolicy Bypass -File .\setup-windal-tunnel.ps1 -Login

Browser opens — pick account that owns zone fasaccountingsoftware.in


STEP 2 — Create tunnel + DNS routes + config.yml (ONE command)
--------------------------------------------------------------
Replace dal-srfipulses with your client key (lowercase, e.g. dal-demo, dal-rgind).

  cd E:\WINDAL\APPTEST
  .\setup-windal-tunnel.cmd dal-srfipulses

Or PowerShell single line:
  powershell -ExecutionPolicy Bypass -File E:\WINDAL\APPTEST\setup-windal-tunnel.ps1 -ClientKey dal-srfipulses

This script:
  - Finds cloudflared (x86 path first)
  - Creates tunnel "dal-srfipulses" OR reuses if it exists
  - Adds DNS:
      dal-srfipulses.fasaccountingsoftware.in
      dal-srfipulses-api.fasaccountingsoftware.in
  - Writes config.yml + <tunnel-uuid>.json in APPTEST folder

Public URLs:
  https://dal-srfipulses.fasaccountingsoftware.in          (web UI, port 5174)
  https://dal-srfipulses-api.fasaccountingsoftware.in      (API, port 5001)


STEP 2b — Routes only (tunnel already exists, fix "No routes")
--------------------------------------------------------------
  .\setup-windal-tunnel.cmd dal-srfipulses routes

Do NOT use Cloudflare dashboard "Add published application" (often HTTP 404).
DNS routes must be created with cloudflared CLI (this script does that).


STEP 3 — Start API + web + tunnel
---------------------------------
All-in-one:
  .\start-windal-stack.cmd

Or three separate terminals:
  npm.cmd run server
  npm.cmd run dev -- --host 0.0.0.0 --port 5174
  .\start-windal-tunnel.cmd

Cloudflare Zero Trust > Tunnels should show:
  Status: Healthy
  Replicas: 1
  Routes: 2 apps


OPTIONAL — Full new PC install (npm + Oracle + scheduled task)
------------------------------------------------------------
Run PowerShell as Administrator:
  powershell -ExecutionPolicy Bypass -File .\setup-windal-tunnel.ps1 -ClientKey dal-srfipulses -FullClientSetup


TROUBLESHOOTING
---------------
- "setup-windal-tunnel.cmd not recognized" → Use .\setup-windal-tunnel.cmd (PowerShell needs .\ for current folder).
- "& $cf not recognized" → You pasted lines in wrong order. Use .\setup-windal-tunnel.cmd instead.
- "cloudflared not valid Win32" → PATH uses System32 stub; use start-windal-tunnel.cmd (correct path).
- Tunnel Inactive / 0 replicas → start-windal-tunnel.cmd not running.
- No routes in dashboard → run: setup-windal-tunnel.cmd <key> routes
- Vite must use port 5174 (Windal default), matching config.yml ingress.
